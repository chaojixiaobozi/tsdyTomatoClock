# Mac 番茄钟 — 产品需求说明（PRD）

## 1. 产品定位与目标

**定位**：在 macOS 笔记本上本地运行的轻量番茄钟，帮助用户建立「专注工作 → 短休 → 长休」的节奏。

**目标**：

- 默认开箱即用，尽量少配置即可开始计时。
- 视觉清晰、动效克制，适合长时间置于副屏或屏幕角落。
- 不依赖账号与云端；数据以本地为主（跨设备同步可作为后续迭代）。

**成功标准（验收导向）**：

- 用户能在约 30 秒内完成「开始第一个番茄」。
- 计时状态一眼可读（剩余时间、当前阶段）。
- 阶段结束时有明确但不骚扰的提醒。

---

## 2. 用户与典型场景

| 角色           | 典型场景                                       |
|----------------|------------------------------------------------|
| 知识工作者     | 深度工作多轮，短休穿插，若干轮后进入长休       |
| 学生           | 自习计时；需暂停/继续以应对临时打断            |
| 多显示器用户   | 窗口可置顶；可缩小为紧凑形态便于瞟一眼         |

---

## 3. 功能需求

### 3.1 核心（MVP，必须实现）

1. **番茄计时**
   - 工作时长、短休、长休可配置；提供合理默认值（建议：25 / 5 / 15 分钟）。
   - 展示剩余时间（大字号为主，可选环形或条形进度辅助）。
   - 控制：**开始 / 暂停 / 继续 / 跳过当前阶段 / 重置本轮**。
   - 「重置」等破坏性操作需二次确认或明确区分，降低误触。

2. **番茄轮次**
   - 完成若干个工作番茄后进入长休（默认每 4 个番茄后长休，可配置）。
   - 界面展示当前阶段：**工作 / 短休 / 长休**，以及可选「第几个番茄」。

3. **到点提醒**
   - 阶段结束：**系统通知**（与 macOS 通知中心行为一致）。
   - 可选：**提示音**（可开关、可选简短铃声）；若产品形态支持，可增加 **Dock 角标或菜单栏状态** 等弱提醒。

4. **会话记录（极简）**
   - 本地记录当日（或本会话）完成的番茄数量，主界面可见。
   - 不要求复杂统计图表或历史多维分析。

5. **应用基础行为**
   - 明确「关闭主窗口」时的语义：**退出应用** 或 **最小化到菜单栏/托盘并保持计时**（二选一，全产品一致实现）。
   - 用户未暂停时，计时在后台应持续运行（与上述窗口策略一致）。

### 3.2 体验增强（建议一期末或 1.1，仍保持简单）

1. **预设方案**：如「经典 25/5」「45/15」「52/17」一键切换。
2. **专注模式**：可选全屏或极简遮罩，减少界面干扰。
3. **快捷键**：开始/暂停、跳过等（可提供默认键位，后续可配置）。
4. **外观**：浅色/深色跟随系统；间距与字体符合 macOS 使用习惯。

### 3.3 明确不做（一期范围外）

- 多人协作、团队看板、社交分享。
- 复杂项目管理（子任务层级、标签体系、甘特图等）。
- 强制联网登录、订阅体系（除非产品方向另行定义）。

## 4. 非功能需求

| 类别   | 要求 |
|--------|------|
| 平台   | 最低 macOS 以《技术方案》为准（**当前为 macOS 12+**，含 Intel / Apple Silicon）；实现栈与 Deployment Target 须一致。 |
| 性能   | 常驻内存与 CPU 占用低；计时误差可接受；实现上优先单调时钟或等价策略，避免明显漂移。 |
| 无障碍 | 主要流程支持键盘操作；颜色对比度满足基础可读性建议。 |
| 隐私   | 默认不上传用户行为数据；若存在本地文件存储，说明路径与内容范围。 |
| 国际化 | 一期可仅中文界面；文案与格式预留扩展英文的可能。 |

---

## 5. 界面与美观（需求级）

1. **信息层级**：剩余时间 > 当前阶段 > 次要操作与配置入口。
2. **视觉**：统一圆角、间距、主色与强调色；状态切换可使用短动画（建议单次 < 300ms）。
3. **状态区分**：进行中、暂停、休息阶段通过颜色或图标弱对比区分，避免刺眼高饱和长时间占屏。
4. **窗口**：默认适中尺寸；支持系统拖拽、全屏/分屏；可选「始终在最前」。

## 6. 验收清单（示例）

- [ ] 默认参数下可完整跑通：工作 → 短休 → 工作 → … → 长休。
- [ ] 暂停后继续，剩余时间与阶段逻辑正确。
- [ ] 阶段结束必达系统通知；通知权限被拒时有清晰引导文案。
- [ ] 若承诺持久化：重启应用后当日完成数（或约定字段）可恢复。
- [ ] 浅色/深色模式下布局无错位、主要文字可读。

---

## 7. 技术形态（PRD 约束）

- 须达到 **原生 Mac 应用级体验或等效体验**（窗口、通知、系统集成）。
- 具体栈（如 Swift/SwiftUI、Tauri、Electron 等）在《技术方案》中选定，并补充：
  - 分发方式（如 DMG、App Store）；
  - 若对外分发：签名与公证等发布要求。
