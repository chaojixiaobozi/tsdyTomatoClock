# Mac 番茄钟 — 产品需求说明（PRD）

## 1. 产品定位与目标

**定位**：在 macOS 笔记本上本地运行的轻量番茄钟，帮助用户建立「专注工作 → 短休 → 长休」的节奏。

**目标**：

- 默认开箱即用，尽量少配置即可开始计时。
- 视觉清晰、动效克制，适合长时间置于副屏或屏幕角落。
- 不依赖账号与云端；数据以本地为主（跨设备同步可作为后续迭代）。

**成功标准（验收导向）**：

- 用户能在约 30 秒内完成「开始第一个番茄」。
- 计时状态一眼可读（剩余时间、当前阶段）。
- 阶段结束时有明确但不骚扰的提醒。

---

## 2. 用户与典型场景

| 角色           | 典型场景                                       |
|----------------|------------------------------------------------|
| 知识工作者     | 深度工作多轮，短休穿插，若干轮后进入长休       |
| 学生           | 自习计时；需暂停/继续以应对临时打断            |
| 多显示器用户   | 窗口可置顶；可缩小为紧凑形态便于瞟一眼         |

---

## 3. 功能需求

### 3.1 核心（MVP，必须实现）

1. **番茄计时**
   - 工作时长、短休、长休可配置；提供合理默认值（建议：25 / 5 / 15 分钟）。
   - 展示剩余时间（大字号为主，可选环形或条形进度辅助）。
   - 控制：**开始 / 暂停 / 继续 / 跳过当前阶段 / 重置本轮**。
   - 「重置」等破坏性操作需二次确认或明确区分，降低误触。

2. **番茄轮次**
   - 完成若干个工作番茄后进入长休（默认每 4 个番茄后长休，可配置）。
   - 界面展示当前阶段：**工作 / 短休 / 长休**，以及可选「第几个番茄」。

3. **到点提醒**
   - 阶段结束：**系统通知**（与 macOS 通知中心行为一致）。
   - 可选：**提示音**（可开关、可选简短铃声）；若产品形态支持，可增加 **Dock 角标或菜单栏状态** 等弱提醒。

4. **会话记录（极简）**
   - 本地记录当日完成的番茄数量，主界面可见。
   - v0.3 起支持按自然日查看历史完成数（月历视图，见第 10 节）；仍不要求复杂统计图表或多维分析。

5. **应用基础行为**
   - 明确「关闭主窗口」时的语义：**退出应用** 或 **最小化到菜单栏/托盘并保持计时**（二选一，全产品一致实现）。
   - 用户未暂停时，计时在后台应持续运行（与上述窗口策略一致）。

### 3.2 体验增强（建议一期末或 1.1，仍保持简单）

1. **预设方案**：如「经典 25/5」「45/15」「52/17」一键切换。
2. **专注模式**：可选全屏或极简遮罩，减少界面干扰。
3. **快捷键**：开始/暂停、跳过等（可提供默认键位，后续可配置）。
4. **外观**：浅色/深色跟随系统；间距与字体符合 macOS 使用习惯。

### 3.3 明确不做（一期范围外）

- 多人协作、团队看板、社交分享。
- 复杂项目管理（子任务层级、标签体系、甘特图等）。
- 强制联网登录、订阅体系（除非产品方向另行定义）。

---

## 4. 非功能需求

| 类别   | 要求 |
|--------|------|
| 平台   | 最低 macOS 以《技术方案》为准（**当前为 macOS 12+**，含 Intel / Apple Silicon）；实现栈与 Deployment Target 须一致。 |
| 性能   | 常驻内存与 CPU 占用低；计时误差可接受；实现上优先单调时钟或等价策略，避免明显漂移。 |
| 无障碍 | 主要流程支持键盘操作；颜色对比度满足基础可读性建议。 |
| 隐私   | 默认不上传用户行为数据；若存在本地文件存储，说明路径与内容范围。 |
| 国际化 | 一期可仅中文界面；文案与格式预留扩展英文的可能。 |

---

## 5. 界面与美观（需求级）

1. **信息层级**：剩余时间 > 当前阶段 > 次要操作与配置入口。
2. **视觉**：统一圆角、间距、主色与强调色；状态切换可使用短动画（建议单次 < 300ms）。
3. **状态区分**：进行中、暂停、休息阶段通过颜色或图标弱对比区分，避免刺眼高饱和长时间占屏。
4. **窗口**：默认适中尺寸；支持系统拖拽、全屏/分屏；可选「始终在最前」。

## 7. 技术形态（PRD 约束）

- 须达到 **原生 Mac 应用级体验或等效体验**（窗口、通知、系统集成）。
- 具体栈（如 Swift/SwiftUI、Tauri、Electron 等）在《技术方案》中选定，并补充：
  - 分发方式（如 DMG、App Store）；
  - 若对外分发：签名与公证等发布要求。

## 9. 新版本需求（v0.2）

本节在 v0.1 基线之上追加；与既有章节冲突时，**以本节为准**。

### 9.1 阶段结束提醒：主窗口前置

- **目标**：在现有弱提醒（系统通知、可选提示音等）之外，增加一层「更容易被注意到」的提醒。
- **行为**：每个阶段计时**自然结束到点**时（与「跳过」区分与否由实现二选一并写入验收说明），将**应用主窗口前置**（激活应用并置于前台，便于用户立即看到当前状态与下一步操作）。
- **约束**：遵守 macOS 窗口与焦点规范；若系统或权限策略导致无法可靠前置，需有等价弱提示（如通知文案中提示「请回到番茄钟确认」），并在《技术方案》中说明。

### 9.2 阶段流转：由「自动进入」改为「手动确认后再进入」

- **现状（v0.1）**：阶段结束后自动进入下一阶段并开始计时（连续流转）。
- **变更（v0.2）**：阶段自然结束后，**暂停在「当前阶段已结束」状态**（剩余时间可为 0 或展示「待确认」类文案，由实现定）；用户**手动确认**（如主按钮「进入下一阶段」或等价交互）后，**再进入下一环节**并开始该段计时。
- **范围**：至少覆盖工作 → 短休/长休、短休 → 工作、长休 → 工作等正常轮次切换；与「跳过」「重置」的交互边界需在实现与验收中写清（例如：跳过是否仍视为需确认，或跳过直接切换）。
- **无障碍**：确认操作须支持键盘可达（与 PRD 第 4 节一致）。

### 9.3 时长预设：一键切换

- **目标**：减少逐项改分钟的成本，提供三种常见节奏的一键应用（在**空闲**或实现允许的安全状态下切换，避免计时中误改参数）。
- **预设表（分钟，可与设置页联动）**：

| 预设名称   | 专注 | 短休 | 长休 | 说明 |
|------------|------|------|------|------|
| 经典 25/5  | 25   | 5    | 15   | 与 v0.1 默认建议一致 |
| 45/15      | 45   | 15   | 15   | 中长专注块 |
| 52/17      | 52   | 17   | 30   | 52/17 节奏；长休略长于短休以便恢复 |

- **轮次规则**：「每几轮后长休」可与 v0.1 相同（默认 4），是否随预设切换一并变化由《技术方案》约定；若不变，须在界面或说明中可读。
- **与第 3.2 节关系**：原「预设方案」体验增强条目由本节**正式纳入** v0.2 范围；实现优先级以本节为准。

### 9.4 验收补充（v0.2）

- [ ] 阶段自然结束时，主窗口前置行为可稳定复现（或与降级策略一致且用户可理解）。
- [ ] 阶段自然结束后，未经手动确认不会自动开始下一阶段计时。
- [ ] 三种预设可一键切换，切换后空闲态展示的各阶段时长与上表一致（或与《技术方案》备案值一致）。

## 10. 新版本需求（v0.3）

本节在 v0.2 基线之上追加；与既有章节冲突时，**以本节为准**。

### 10.1 按天历史记录：番茄完成数

- **目标**：在现有「当日完成数」基础上，本地保留**按自然日**汇总的番茄完成次数，便于用户回顾近期专注节奏；仍保持极简，不做复杂分析。
- **计数口径**：与 v0.1 一致——**仅在工作阶段自然完成或经产品已定义的等价完成路径**时 +1（与 Domain 现有「完成工作阶段 → 番茄计数 +1」规则对齐；跳过是否计数以《技术方案》与 Domain 单测为准，须与当日计数一致）。
- **自然日**：以用户系统时区下的**日历日**为键（与现有 `YYYY-MM-DD` 日字符串语义一致）；跨日零点起算新一天，当日计数从 0 开始，**前一日最终完成数写入历史**。
- **持久化**：
  - 数据仅存本地（与 PRD 隐私要求一致），默认不上传。
  - 每条记录至少包含：**日期**、**当日完成番茄数**；存储格式与保留策略由《技术方案》约定（如 UserDefaults 字典 / JSON，保留最近 N 天或全量本地存储并设合理上限）。
- **展示**：
  - 主界面仍突出**今日完成数**（行为与 v0.1/v0.2 一致）。
  - **入口位置**：主界面与设置同级的操作区，**历史**入口位于**设置**入口的**左侧**（从左到右：历史 → 设置）。
  - **视图形态**：以**月历**展示，每次呈现**一个自然月**的全部日期格；每格展示该日日期与当日完成的番茄数（同格层级由实现定，须一眼可读）。
  - **月份切换**：支持向**上一月 / 下一月**切换（如顶部箭头、按钮或等价交互）；默认打开为**当前自然月**。
  - 当月无记录日可显示为 0 或留空，由实现二选一并保持一致；**不要求**折线图、热力图、周/月聚合汇总行等多维统计。
- **边界**：
  - 应用重启、系统休眠后，历史与当日计数须可恢复且不与跨日逻辑冲突。
  - 用户修改系统日期/时区时，以《技术方案》说明的合并或截断策略为准，界面不出现无法理解的重复或丢失（可接受降级为「自某日起重新累计」并文档化）。

### 10.2 明确仍不做（v0.3 范围外）

- 导出 CSV、iCloud 同步、跨设备合并。
- 按任务/标签/项目维度的番茄归因。
- 专注时长（分钟）统计、中断次数、效率评分等扩展指标。

### 10.3 验收补充（v0.3）

- [ ] 连续两天各完成若干番茄后，历史月历中对应两日格内完成数分别正确；第三日 0 点或跨日后「今日」从 0 起算。
- [ ] 重启应用后，今日计数与已写入历史的各日数据与重启前一致。
- [ ] 历史月历可读、键盘可达（与第 4 节无障碍一致）；可切换至相邻月份并正确展示该月各日数据；无记录日为空态或 0，文案清晰。
- [ ] 历史仅本地存储，无联网上报（可通过代码审查或《技术方案》键名说明验收）。
